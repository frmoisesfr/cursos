1. ¿Qué es el reconocimiento facial?

Imagina que tu móvil se desbloquea al mirarte, o que una cámara sabe quién entra en un edificio. Eso es reconocimiento facial: la capacidad de un ordenador para detectar e identificar caras humanas.

🧠 Funciona en 3 pasos

1. Detección: Encontrar dónde hay una cara.

2. Análisis: Medir sus características (distancia entre ojos, forma de nariz…).

3. Reconocimiento: Comparar con caras conocidas.

🧠 ¿Por qué importa?

Si cargas una imagen con cv2.imread() (BGR) y la pasas directamente a face_recognition (que espera RGB), los colores estarán invertidos. Por eso usamos cv2.cvtColor(img, cv2.COLOR_BGR2RGB) para convertir.

7. Puntos faciales (Landmarks)

Los landmarks (puntos de referencia) son 68 puntos distribuidos por la cara que mapean su estructura: contorno de la mandíbula (17 puntos), cejas (5+5), nariz (9), ojos (6+6) y boca (20). Son la base para que el ordenador "entienda" la geometría de un rostro.

import cv2, face_recognition

imagen = face_recognition.load_image_file("foto.jpg")
lista_landmarks = face_recognition.face_landmarks(imagen)
imagen_bgr = cv2.cvtColor(imagen, cv2.COLOR_RGB2BGR)

colores = {"chin":(255,255,255), "left_eyebrow":(0,255,255), "right_eyebrow":(0,255,255),
           "nose_bridge":(0,255,0), "nose_tip":(0,255,0), "left_eye":(255,0,0),
           "right_eye":(255,0,0), "top_lip":(0,0,255), "bottom_lip":(0,0,255)}

for landmarks in lista_landmarks:
    for parte, puntos in landmarks.items():
        color = colores.get(parte, (255,255,255))

            cv2.circle(imagen_bgr, (x, y), 2, color, -1)

8. Comparar dos caras

🧠 Encoding = huella digital numérica de la cara

El sistema usa una red neuronal convolucional (CNN) entrenada con millones de rostros para convertir cada cara en un vector de 128 números decimales. Estos números codifican características abstractas aprendidas automáticamente: proporciones faciales, distancia entre ojos, forma de la mandíbula, etc. Dos fotos de la misma persona, incluso con diferente iluminación o ángulo, producen vectores cercanos. Personas distintas producen vectores lejanos. La "distancia" entre vectores se mide con la distancia euclidiana.

comparar_caras.py
import face_recognition

img_a = face_recognition.load_image_file("persona_a.jpg")
img_b = face_recognition.load_image_file("persona_b.jpg")

enc_a = face_recognition.face_encodings(img_a)[0]
enc_b = face_recognition.face_encodings(img_b)[0]

resultado = face_recognition.compare_faces([enc_a], enc_b)
distancia = face_recognition.face_distance([enc_a], enc_b)

if resultado[0]:
    print("✅ ¡Misma persona!")
else:
    print("❌ Personas diferentes.")
print(f"📏 Distancia: {distancia[0]:.4f}")
face_distance()
Calcula la distancia euclidiana entre dos encodings. 0.0 = caras idénticas. < 0.4 = casi seguro la misma persona. 0.4–0.6 = zona dudosa. > 0.6 = personas distintas. El umbral por defecto de compare_faces() es 0.6, pero en nuestro proyecto usaremos 0.5 para reducir falsos positivos.

webcam_caras.py
import cv2
import face_recognition

camara = cv2.VideoCapture(0)
print("📷 Pulsa 'q' para salir.")

while True:
    ret, frame = camara.read()
    if not ret: break

    # Reducir al 25% para mayor velocidad de detección
    pequeno = cv2.resize(frame, (0,0), fx=0.25, fy=0.25)
    rgb = cv2.cvtColor(pequeno, cv2.COLOR_BGR2RGB)

    # Detectar caras en la imagen reducida
    ubicaciones = face_recognition.face_locations(rgb)

    # Dibujar rectángulos (coordenadas × 4 porque redujimos)
    for (arriba, derecha, abajo, izquierda) in ubicaciones:
        arriba *= 4
        derecha *= 4
        abajo *= 4
        izquierda *= 4
        cv2.rectangle(frame, (izquierda, arriba), (derecha, abajo), (0,255,0), 2)

    cv2.imshow("Webcam - Detección facial", frame)
    if cv2.waitKey(1) & 0xFF == ord("q"):
        break

camara.release()
cv2.destroyAllWindows()
while True
Bucle infinito. Se repite continuamente: captura un fotograma, busca caras y muestra el resultado. Un vídeo no es más que una sucesión de fotos (fotogramas o frames) mostradas muy rápido, unas ~30 por segundo.
resize(fx=0.25)
Reduce la imagen al 25% de su tamaño original. La detección facial es computacionalmente costosa: al trabajar con una imagen 16 veces más pequeña (4×4), el proceso se acelera drásticamente. Luego multiplicamos las coordenadas ×4 para dibujar en la posición correcta sobre el frame original.
& 0xFF
Operación AND a nivel de bits que asegura compatibilidad entre sistemas operativos al leer el código de la tecla pulsada. ord("q") convierte el carácter "q" a su valor numérico ASCII (113).

10. Registrar personas conocidas

registrar.py
import os, face_recognition

def cargar_personas(carpeta):
    nombres, encodings = [], []
    for f in os.listdir(carpeta):
        if not f.lower().endswith((".jpg",".jpeg",".png")): continue
        nombre = os.path.splitext(f)[0]
        img = face_recognition.load_image_file(os.path.join(carpeta, f))
        enc = face_recognition.face_encodings(img)
        if enc:
            encodings.append(enc[0]); nombres.append(nombre)
            print(f"  ✅ {nombre}")
    return nombres, encodings

nombres, encodings = cargar_personas("personas_conocidas")
print(f"🎉 {len(nombres)} registrada(s)")
def
Define una función: un bloque de código reutilizable con nombre y parámetros. Encapsulas la lógica una vez y la invocas tantas veces como necesites con cargar_personas("carpeta"). Esto es un principio fundamental de programación: no repitas código (DRY — Don't Repeat Yourself).

11. Manejo de errores con try/except

Hasta ahora, si algo falla (la foto no existe, la cámara está ocupada, no se detecta ninguna cara), nuestros programas se interrumpen con un error. Python ofrece un mecanismo llamado manejo de excepciones que permite capturar estos errores y reaccionar de forma controlada.

🧠 try / except / finally

try ejecuta un bloque de código que podría fallar. Si se produce una excepción (error), Python salta al bloque except correspondiente en vez de detenerse. Opcionalmente, finally se ejecuta siempre, haya error o no (útil para liberar recursos como archivos o la cámara).

errores_controlados.py
import cv2, face_recognition

def cargar_imagen_segura(ruta):
    try:
        return face_recognition.load_image_file(ruta)
    except FileNotFoundError:
        print(f"❌ No existe: {ruta}")
        return None
    except Exception as e:
        print(f"⚠️ Error: {e}")
        return None

def encoding_seguro(imagen):
    try:
        enc = face_recognition.face_encodings(imagen)
        if not enc:
            print("⚠️ No se detectó cara")
            return None
        return enc[0]
    except Exception as e:
        print(f"⚠️ Error: {e}")
        return None

def abrir_camara_segura():
    cam = cv2.VideoCapture(0)
    if not cam.isOpened():
        print("❌ Cámara no disponible.")
        print("   Cierra Zoom/Meet y reintenta.")
        return None
    return cam
try
Bloque de código que podría fallar. Python lo "vigila".
except
Si hay error, ejecuta este bloque en vez de romperse. Puedes capturar errores específicos (FileNotFoundError) o genéricos (Exception).

import cv2
import face_recognition
import os
import numpy as np


def cargar_personas(carpeta):
    """Carga fotos de una carpeta y genera encodings."""
    nombres = []
    encodings = []

    if not os.path.exists(carpeta):
        print(f"❌ Carpeta '{carpeta}' no encontrada")
        return nombres, encodings

    for archivo in os.listdir(carpeta):
        if not archivo.lower().endswith((".jpg", ".jpeg", ".png")):
            continue
        try:
            ruta = os.path.join(carpeta, archivo)
            imagen = face_recognition.load_image_file(ruta)
            lista_enc = face_recognition.face_encodings(imagen)
            if lista_enc:
                encodings.append(lista_enc[0])
                nombre = os.path.splitext(archivo)[0]
                nombres.append(nombre)
                print(f"  ✅ {nombre}")
        except Exception as error:
            print(f"  ⚠️ Error con {archivo}: {error}")

    return nombres, encodings


# ─── Cargar base de datos ───
nombres_bd, encodings_bd = cargar_personas("personas_conocidas")
print(f"🎉 {len(nombres_bd)} persona(s) cargada(s)\n")

# ─── Abrir cámara ───
camara = cv2.VideoCapture(0)
print("📷 Pulsa 'q' para salir.")

contador_frames = 0
nombres_actuales = []
ubicaciones_actuales = []

while True:
    ret, frame = camara.read()
    if not ret:
        break

    contador_frames += 1

    # Solo procesar cada 3 frames (optimización)
    if contador_frames % 3 == 0:
        pequeno = cv2.resize(frame, (0, 0), fx=0.25, fy=0.25)
        rgb = cv2.cvtColor(pequeno, cv2.COLOR_BGR2RGB)

        ubicaciones_actuales = face_recognition.face_locations(rgb)
        encodings_frame = face_recognition.face_encodings(rgb, ubicaciones_actuales)

        nombres_actuales = []
        for encoding in encodings_frame:
            if encodings_bd:
                distancias = face_recognition.face_distance(encodings_bd, encoding)
                indice_menor = np.argmin(distancias)
                if distancias[indice_menor] < 0.5:
                    nombre = nombres_bd[indice_menor]
                else:
                    nombre = "Desconocido"
            else:
                nombre = "Sin datos"
            nombres_actuales.append(nombre)

    # Dibujar resultados (en cada frame para fluidez)
    for (arriba, derecha, abajo, izquierda), nombre in zip(ubicaciones_actuales, nombres_actuales):
        # Escalar coordenadas ×4
        arriba *= 4
        derecha *= 4
        abajo *= 4
        izquierda *= 4

        color = (0, 255, 0) if nombre != "Desconocido" else (0, 0, 255)
        cv2.rectangle(frame, (izquierda, arriba), (derecha, abajo), color, 2)
        cv2.rectangle(frame, (izquierda, abajo - 28), (derecha, abajo), color, cv2.FILLED)
        cv2.putText(frame, nombre, (izquierda + 6, abajo - 6),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.6, (255, 255, 255), 1)

    cv2.imshow("Reconocimiento Facial", frame)
    if cv2.waitKey(1) & 0xFF == ord("q"):
        break

camara.release()
cv2.destroyAllWindows()

13. Detectar emociones (simplificado)

Vamos a analizar la geometría de la boca para estimar si una persona sonríe. La idea es calcular el ratio entre el ancho y la altura de la boca: cuando sonreímos, la boca se ensancha más de lo que se abre.

ℹ️ Versión simplificada

Los sistemas profesionales (como deepface o fer) usan redes neuronales entrenadas con miles de expresiones faciales. Aquí usamos geometría básica para entender el concepto. Es una aproximación, no un sistema fiable.

detectar_sonrisa.py
import cv2
import face_recognition
import numpy as np


def analizar_sonrisa(landmarks):
    """Estima si hay sonrisa según el ratio ancho/alto de la boca."""
    try:
        puntos_labio_sup = np.array(landmarks["top_lip"])
        puntos_labio_inf = np.array(landmarks["bottom_lip"])

        # Ancho: distancia entre comisuras izquierda y derecha
        ancho_boca = np.linalg.norm(puntos_labio_sup[6] - puntos_labio_sup[0])

        # Alto: distancia entre centro del labio superior e inferior
        alto_boca = np.linalg.norm(puntos_labio_inf[3] - puntos_labio_sup[3])

        if alto_boca == 0:
            return "😐 Neutro"

        ratio = ancho_boca / alto_boca

        # Ratio alto = boca ancha y poco abierta = sonrisa
        if ratio > 3.0:
            return "😁 ¡Sonrisa!"
        elif ratio > 2.0:
            return "🙂 Sonrisa leve"
        else:
            return "😐 Neutro"
    except Exception:
        return "❓ Sin datos"


camara = cv2.VideoCapture(0)
print("📷 Detector de sonrisas. Pulsa 'q' para salir.")

while True:
    ret, frame = camara.read()
    if not ret:
        break

    # Reducir al 50% (necesitamos más resolución para landmarks)
    pequeno = cv2.resize(frame, (0, 0), fx=0.5, fy=0.5)
    rgb = cv2.cvtColor(pequeno, cv2.COLOR_BGR2RGB)

    ubicaciones = face_recognition.face_locations(rgb)
    lista_landmarks = face_recognition.face_landmarks(rgb, ubicaciones)

    for (arriba, derecha, abajo, izquierda), landmarks in zip(ubicaciones, lista_landmarks):
        emocion = analizar_sonrisa(landmarks)

        # Escalar ×2 (redujimos al 50%)
        arriba *= 2
        derecha *= 2
        abajo *= 2
        izquierda *= 2

        cv2.rectangle(frame, (izquierda, arriba), (derecha, abajo), (255, 200, 0), 2)
        cv2.putText(frame, emocion, (izquierda, arriba - 10),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.7, (255, 200, 0), 2)

    cv2.imshow("Detector de sonrisas", frame)
    if cv2.waitKey(1) & 0xFF == ord("q"):
        break

camara.release()
cv2.destroyAllWindows()
np.linalg.norm()
Calcula la distancia euclidiana entre dos puntos (la fórmula del teorema de Pitágoras generalizada). linalg = álgebra lineal. norm = norma (longitud) de un vector. Al restar dos arrays NumPy se obtiene el vector entre ambos puntos.
fx=0.5
Para los landmarks usamos 50% en vez de 25% porque la detección de puntos faciales necesita más resolución que la simple detección de caras. Es un compromiso entre velocidad y precisión.

import cv2
import face_recognition
import os
import numpy as np
from datetime import datetime


def cargar_personas(carpeta):
    """Carga todas las fotos de la carpeta y genera encodings."""
    nombres = []
    encodings = []

    if not os.path.exists(carpeta):
        print(f"❌ Carpeta '{carpeta}' no encontrada")
        return nombres, encodings

    for archivo in os.listdir(carpeta):
        if not archivo.lower().endswith((".jpg", ".jpeg", ".png")):
            continue
        try:
            ruta = os.path.join(carpeta, archivo)
            imagen = face_recognition.load_image_file(ruta)
            lista_enc = face_recognition.face_encodings(imagen)
            if lista_enc:
                encodings.append(lista_enc[0])
                nombres.append(os.path.splitext(archivo)[0])
                print(f"  ✅ {nombres[-1]}")
        except Exception as error:
            print(f"  ⚠️ Error con {archivo}: {error}")

    return nombres, encodings


def registrar_asistencia(nombre, archivo_csv):
    """Registra asistencia en CSV evitando duplicados del mismo día."""
    ahora = datetime.now()
    fecha = ahora.strftime("%Y-%m-%d")
    hora = ahora.strftime("%H:%M:%S")

    # Comprobar si ya se registró hoy
    registrados_hoy = set()
    if os.path.exists(archivo_csv):
        with open(archivo_csv, "r") as fichero:
            for linea in fichero:
                partes = linea.strip().split(",")
                if len(partes) >= 2 and partes[1] == fecha:
                    registrados_hoy.add(partes[0])

    if nombre not in registrados_hoy:
        with open(archivo_csv, "a") as fichero:
            fichero.write(f"{nombre},{fecha},{hora}\n")

    return False

    nombre_archivo = f"{nombre}_{marca_tiempo}.jpg"
    ruta = os.path.join(carpeta_capturas, nombre_archivo)
    cv2.imwrite(ruta, frame)

🟣 Extra · Teoría

🔬 ¿Cómo funciona HOG por dentro?

Cuando llamamos a face_locations(), internamente se ejecuta el algoritmo HOG (Histogram of Oriented Gradients). Vamos a entender qué hace paso a paso:

Paso 1: Convertir a escala de grises

El color no aporta información útil para detectar formas. HOG trabaja solo con brillo (blanco y negro), lo que reduce los datos a un tercio.

Paso 2: Calcular gradientes

Un gradiente es la diferencia de brillo entre dos píxeles vecinos. Si un píxel es oscuro y su vecino es claro, hay un gradiente fuerte → eso es un borde. HOG calcula la dirección (horizontal, vertical, diagonal) y la magnitud de cada gradiente.

🧠 Analogía

Imagina que pasas la mano por una superficie con los ojos cerrados. Donde notas un cambio brusco (de liso a rugoso), hay un "borde". HOG hace exactamente eso, pero comparando números de brillo entre píxeles vecinos. Un cambio brusco de 20 a 200 = borde fuerte. Un cambio de 100 a 105 = zona uniforme.

Paso 3: Histogramas por regiones

La imagen se divide en celdas de 8×8 píxeles. En cada celda se crea un histograma (un gráfico de barras) que cuenta cuántos gradientes apuntan en cada dirección. Esto captura la "textura" de cada zona sin depender de la posición exacta de cada píxel.

Paso 4: Comparar con el patrón "cara"

El modelo HOG preentrenado sabe qué aspecto tiene una cara típica en gradientes: zona oscura en los ojos, bordes verticales en la nariz, bordes horizontales en la boca. Desliza una ventana por toda la imagen y compara cada región con ese patrón. Donde hay coincidencia → cara detectada.

💡 HOG vs CNN

HOG es rápido (funciona en CPU) pero menos preciso con caras de perfil o parcialmente ocultas. CNN (model="cnn") usa una red neuronal: mucho más precisa pero necesita GPU. Para proyectos escolares, HOG es más que suficiente.

⚡ Alternativa moderna: MediaPipe

MediaPipe es la biblioteca de Google para visión por computador. Es mucho más rápida que face_recognition (funciona a 30+ FPS sin reducir la imagen) y detecta 468 landmarks en vez de 68. Aquí tienes un ejemplo mínimo para que veas la diferencia:

import cv2
import mediapipe as mp

# Inicializar el detector de MediaPipe
mp_face = mp.solutions.face_mesh
mp_dibujo = mp.solutions.drawing_utils

detector = mp_face.FaceMesh(
    max_num_faces=3,           # Detectar hasta 3 caras
    min_detection_confidence=0.5
)

camara = cv2.VideoCapture(0)
print("📷 MediaPipe FaceMesh · Pulsa 'q' para salir")

while True:
    ret, frame = camara.read()
    if not ret:
        break

    # MediaPipe necesita RGB (igual que face_recognition)
    rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    resultados = detector.process(rgb)

    # Dibujar los 468 landmarks si se detectó cara
    if resultados.multi_face_landmarks:
        for cara in resultados.multi_face_landmarks:
            mp_dibujo.draw_landmarks(
                frame, cara,
                mp_face.FACEMESH_TESSELATION,  # Malla triangular
                landmark_drawing_spec=None,
                connection_drawing_spec=mp_dibujo.DrawingSpec(
                    color=(0, 255, 0), thickness=1
                )
            )

    cv2.imshow("MediaPipe FaceMesh", frame)
    if cv2.waitKey(1) & 0xFF == ord("q"):
        break

camara.release()
cv2.destroyAllWindows()
🧠 face_recognition vs MediaPipe

face_recognition es ideal para aprender porque su API es simple y sus funciones son intuitivas. MediaPipe es lo que usarías en un proyecto real: 468 landmarks, detección de manos y pose corporal, funciona en móviles, y no necesita reducir la imagen. El paso natural después de este curso es migrar tu proyecto a MediaPipe.
