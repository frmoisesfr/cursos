1. ¿Qué es el reconocimiento facial?

Tu móvil se desbloquea al mirarte. Los filtros de TikTok te ponen orejas de gato. Las cámaras de un aeropuerto saben quién eres sin que enseñes el pasaporte. Todo esto es reconocimiento facial: la capacidad de un ordenador para encontrar caras en una imagen y saber de quién son.

🧠 El proceso en 3 pasos (simplificado)

1. Detectar: El ordenador encuentra dónde hay una cara en la imagen. Es como cuando tú miras una foto de grupo y tus ojos van directos a las caras.

2. Analizar: Mide las características de esa cara (distancia entre ojos, forma de la nariz, contorno de la mandíbula…) y las convierte en una lista de números. Es como el "DNI numérico" de esa cara.

3. Comparar: Compara esos números con los de caras que ya conoce. Si los números son parecidos → es la misma persona.

3. Tu primer programa: analizar una cara

Vamos a hacer algo espectacular en pocas líneas: darle una foto a DeepFace y que nos diga la emoción, la edad y el género de la persona. Primero sube una foto con una cara visible (sección anterior) y luego ejecuta este código:

Celda — Tu primer análisis facial
from deepface import DeepFace

# ─── Analizar la foto ───
# Cambia "foto.jpg" por el nombre de tu foto subida
resultados = DeepFace.analyze(
    img_path="foto.jpg",
    actions=["age", "gender", "emotion"]
)

# ─── Mostrar lo que ha descubierto ───
for cara in resultados:
    print("═══════════════════════════════")
    print(f"😊 Emoción detectada: {cara['dominant_emotion']}")
    print(f"🎂 Edad estimada:     {cara['age']} años")
    print(f"👤 Género:            {cara['dominant_gender']}")
    print(f"📍 Posición cara:     x={cara['region']['x']}, y={cara['region']['y']}")
    print(f"📐 Tamaño cara:       {cara['region']['w']}×{cara['region']['h']} píxeles")
    print("═══════════════════════════════")
DeepFace.analyze()
Es la función principal de DeepFace. Le das una foto y te devuelve información sobre cada cara que encuentre. El parámetro actions le dice qué quieres saber: "age" = edad, "gender" = género, "emotion" = emoción. También puedes pedir "race" (etnia).
resultados
Es una lista. Si la foto tiene 1 cara, la lista tiene 1 elemento. Si tiene 3 caras, tiene 3 elementos. Cada elemento es un diccionario (como una ficha) con toda la información de esa cara: edad, emoción, género, posición, etc.
cara['region']
Contiene las coordenadas de la cara dentro de la imagen: x (posición horizontal), y (posición vertical), w (ancho) y h (alto). Estos números se usan luego para dibujar rectángulos alrededor de las caras.

4. Detectar y extraer caras de una foto

En la sección anterior, DeepFace detectaba caras automáticamente al analizarlas. Ahora vamos a usar una función específica para encontrar todas las caras de una imagen y obtener información sobre cada una.

Celda — Detectar caras
from deepface import DeepFace

# ─── Buscar todas las caras en la imagen ───
caras = DeepFace.extract_faces(
    img_path="foto_grupo.jpg",
    detector_backend="opencv",  # El detector mas rapido
    align=True                   # Alinear caras (ojos horizontales)
)

# ─── Mostrar cuántas caras encontró ───
print(f"🔍 Caras encontradas: {len(caras)}")

# ─── Detalles de cada cara ───
for numero, cara in enumerate(caras, start=1):
    zona = cara["facial_area"]       # Coordenadas de la cara
    confianza = cara["confidence"]  # Seguridad de la detección (0 a 1)

    print(f"\n  👤 Cara {numero}:")
    print(f"     Posición: x={zona['x']}, y={zona['y']}")
    print(f"     Tamaño:   {zona['w']}×{zona['h']} píxeles")
    print(f"     Confianza: {confianza:.2%}")
extract_faces()
Busca todas las caras de una imagen y devuelve una lista. Cada elemento contiene: la imagen de la cara recortada (face), las coordenadas (facial_area), y la confianza de detección (confidence). Es diferente de analyze(): esta función solo encuentra caras, no las analiza.
detector_backend
Le dice a DeepFace qué algoritmo usar para buscar caras. "opencv" es el más rápido y sencillo. "retinaface" es el más preciso (detecta caras de perfil y parcialmente tapadas). "mtcnn" es un buen equilibrio. Para empezar, "opencv" va perfecto.
enumerate(caras, start=1)
enumerate() es una función de Python que recorre una lista y te da un número junto con cada elemento. start=1 hace que empiece a contar desde 1 (en vez de 0). Así podemos imprimir "Cara 1", "Cara 2", etc.
:.2%
Formato de Python que convierte un número decimal en porcentaje con 2 decimales. Ejemplo: 0.9834 se muestra como 98.34%. Los : introducen el formato y .2% indica "2 decimales y símbolo %".

5. Dibujar rectángulos y etiquetas sobre las caras

Ahora vamos a combinar la detección con el dibujo: DeepFace encuentra las caras y sus emociones, y nosotros dibujamos un rectángulo verde y una etiqueta encima de cada una. Así podrás crear tus propias imágenes de "reconocimiento facial" como las que ves en las películas.

Celda — Dibujar sobre las caras
from deepface import DeepFace
import cv2
from google.colab.patches import cv2_imshow

# ─── Paso 1: Cargar la imagen con OpenCV ───
imagen = cv2.imread("foto_grupo.jpg")

# ─── Paso 2: Analizar todas las caras ───
resultados = DeepFace.analyze(
    img_path="foto_grupo.jpg",
    actions=["emotion", "age"],
    detector_backend="retinaface",  # Detecta caras de perfil y parcialmente ocultas
    enforce_detection=False          # No dar error si alguna cara no se detecta
)

# ─── Paso 3: Dibujar un rectángulo y etiqueta en cada cara ───
for cara in resultados:
    # Obtener las coordenadas de la cara
    x = cara["region"]["x"]
    y = cara["region"]["y"]
    ancho = cara["region"]["w"]
    alto = cara["region"]["h"]

    # Preparar el texto de la etiqueta
    emocion = cara["dominant_emotion"]
    edad = cara["age"]
    etiqueta = f"{emocion}, {edad} year"

    # Dibujar rectángulo verde alrededor de la cara
    cv2.rectangle(imagen, (x, y), (x + ancho, y + alto), (0, 255, 0), 2)

    # Escribir la etiqueta encima del rectángulo
    cv2.putText(imagen, etiqueta, (x, y - 10),
                cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0, 255, 0), 2)

# ─── Paso 4: Mostrar el resultado ───
cv2_imshow(imagen)

# ─── Paso 5: Guardar la imagen resultante ───
cv2.imwrite("resultado.jpg", imagen)
print("💾 Imagen guardada como resultado.jpg")
cv2.rectangle()
Dibuja un rectángulo. Los parámetros son: imagen donde dibujar, esquina superior izquierda (x, y), esquina inferior derecha (x+ancho, y+alto), color (0, 255, 0) = verde en formato BGR, y grosor de línea 2 píxeles.
cv2.putText()
Escribe texto sobre la imagen. Parámetros: imagen, texto, posición (x, y-10) (un poco por encima del rectángulo), tipo de letra, tamaño 0.6, color verde, y grosor 2.
detector_backend
Usamos "retinaface" porque es el detector más preciso: detecta caras de perfil, parcialmente ocultas y en ángulo. El detector por defecto ("opencv") solo funcióna bien con caras de frente. En fotos de grupo, la diferencia es enorme. Tarda un poco más, pero merece la pena.
enforce_detection=False
Por defecto, DeepFace da error si no encuentra cara. Con False, simplemente ignora las zonas donde no hay cara. Útil en fotos de grupo donde alguna cara puede estar parcialmente oculta.
cv2.imwrite()
Guarda la imagen modificada como un archivo nuevo. En Colab, el archivo se guarda en el espacio temporal del servidor. Puedes descargarlo con from google.colab import files; files.download("resultado.jpg").

6. Verificar: ¿son la misma persona?

Imagina que tienes la foto del DNI de alguien y una foto actual. ¿Son la misma persona? DeepFace puede responder a esa pregunta con una sola función. Sube dos fotos a Colab y prueba:

Celda — Comparar dos caras
from deepface import DeepFace

# ─── Comparar dos fotos ───
# Sube dos fotos primero con files.upload()
resultado = DeepFace.verify(
    img1_path="foto_persona_a.jpg",
    img2_path="foto_persona_b.jpg"
)

# ─── ¿Son la misma persona? ───
if resultado["verified"]:
    print("✅ ¡Son la MISMA persona!")
else:
    print("❌ Son personas DIFERENTES.")

print(f"\n📏 Distancia numérica: {resultado['distance']:.4f}")
print(f"📊 Umbral de decisión: {resultado['threshold']:.4f}")
print(f"🧠 Modelo usado:       {resultado['model']}")

if resultado["distance"] < resultado["threshold"]:
    print(f"\n💡 La distancia ({resultado['distance']:.4f}) es MENOR que el umbral ({resultado['threshold']:.4f})")
    print("   → Por eso decide que SÍ son la misma persona.")
else:
    print(f"\n💡 La distancia ({resultado['distance']:.4f}) es MAYOR que el umbral ({resultado['threshold']:.4f})")
    print("   → Por eso decide que NO son la misma persona.")
DeepFace.verify()
Compara dos caras y dice si son la misma persona. Devuelve un diccionario con: verified (True o False), distance (qué tan diferentes son los números de ambas caras), y threshold (el límite de decisión). Si la distancia es menor que el umbral → misma persona.
distance
Es un número que mide lo "lejos" que están las dos caras en términos matemáticos. Cuanto más pequeño, más parecidas son las caras. Un valor de 0.0 significaría que son exactamente idénticas (imposible en la práctica). Valores pequeños (menores que el umbral) = misma persona. Valores grandes = personas diferentes.
threshold
Es el valor de corte que usa DeepFace para decidir. Cada modelo de IA tiene su propio umbral. Por ejemplo, VGG-Face usa ~0.40. Si la distancia es 0.25 (menor que 0.40) → misma persona. Si es 0.65 (mayor que 0.40) → diferentes.

7. Análisis completo: edad, género, emoción

Ya vimos DeepFace.analyze() en la sección 3. Ahora vamos a profundizar: veremos las probabilidades de cada emoción (no solo la dominante) y crearemos un gráfico de barras visual directamente en la consola.

Celda — Análisis detallado con barras de emoción
from deepface import DeepFace

resultados = DeepFace.analyze(
    img_path="foto.jpg",
    actions=["age", "gender", "emotion"],
    enforce_detection=False
)

for numero, cara in enumerate(resultados, start=1):
    print(f"\n══════ Cara {numero} ══════")
    print(f"🎂 Edad:    {cara['age']} años")
    print(f"👤 Género:  {cara['dominant_gender']}")
    print(f"😊 Emoción: {cara['dominant_emotion']}")

    # Mostrar TODAS las emociones con barras visuales
    print("\n   Probabilidad de cada emocion:")
    for emocion, probabilidad in cara["emotion"].items():
        # Crear una barra visual con bloques ██
        num_bloques = int(probabilidad / 5)  # 1 bloque por cada 5%
        barra = "█" * num_bloques
        print(f"   {emocion:>10}: {barra} {probabilidad:.1f}%")
cara["emotion"]
Además de la emoción dominante, DeepFace devuelve un diccionario con la probabilidad de cada una de las 7 emociones: angry (enfado), disgust (asco), fear (miedo), happy (alegría), sad (tristeza), surprise (sorpresa), neutral. Cada valor es un porcentaje de 0 a 100. Por ejemplo: 72% happy, 15% neutral, 8% surprise, 5% el resto.
"█" * num_bloques
En Python, multiplicar un texto por un número lo repite. "█" * 5 genera "█████". Dividimos la probabilidad entre 5 para que una emoción con 50% tenga 10 bloques, una con 100% tenga 20, etc. Es un "gráfico de barras" hecho con caracteres de texto.
{emoción:>10}
Formato que alínea el texto a la derecha en un espacio de 10 caracteres. Así todas las barras empiezan en la misma columna, independientemente de si la emoción se llama "sad" (3 letras) o "surprise" (8 letras). Es un truco para que la salida sea visualmente ordenada.

from google.colab import files
import os

resultados = DeepFace.find(
    img_path=foto_desconocida,
    db_path="base_datos",
    enforce_detection=False,
    silent=True
)

# ─── Interpretar el resultado ───
if len(resultados) > 0 and len(resultados[0]) > 0:
    mejor_coincidencia = resultados[0].iloc[0]
    ruta_foto = mejor_coincidencia["identity"]
    distancia = mejor_coincidencia["distance"]

    # Extraer el nombre del archivo (sin extensión ni carpeta)

    print(f"📄 Foto coincidente: {ruta_foto}")

    print("❓ No se encontró coincidencia en la base de datos.")
    print("   Puede ser una persona no registrada o la foto no es clara.")
DeepFace.find()
Busca la cara de la imagen en todas las fotos de la carpeta db_path. Devuelve una lista de coincidencias ordenadas de mejor a peor (menor distancia = más parecido). La primera vez tarda más porque genera un archivo de caché con los "números" (embeddings) de todas las caras. Las veces siguientes es mucho más rápido.
.iloc[0]
iloc es un método de pandas (biblioteca de datos) que accede a una fila por posición. iloc[0] selecciona la primera fila = la coincidencia más cercana. Es como decir "dame el resultado número 1 del ranking".
os.path.basename()
De una ruta como "base_datos/Carlos.jpg", extrae solo el nombre del archivo: "Carlos.jpg". Luego splitext() separa el nombre de la extensión, quedándonos con "Carlos".

DeepFace.stream()
Abre la webcam, detecta caras, las compara con la base de datos, y muestra en pantalla el nombre, emoción, edad y género. Todo automático. Pulsa Q para salir. Es la forma mas rápida de tener un sistema de reconocimiento facial completo funciónando.
db_path
Ruta a la carpeta dónde están las fotos de personas conocidas. Puede ser una ruta relativa (como "base_datos", que busca dentro de tu carpeta de proyecto) o una ruta completa (como "C:\Users\tu_nombre\fotos").

Parámetros que puedes modificar

La función stream() acepta mas parametros para personalizar su comportamiento:

from deepface import DeepFace

DeepFace.stream(
    db_path="base_datos",
    source=0,
    model_name="VGG-Face",          # Modelo de reconocimiento (ver seccion 12)

anti_spoofing

webcam_personalizada.py
from deepface import DeepFace
import cv2
import os

# ═══ CONFIGURACION ═══
CARPETA_BD = "base_datos"     # Carpeta con fotos conocidas
FRECUENCIA = 45               # Analizar cada X frames (~1.5 seg a 30fps)

# ═══ Abrir la webcam ═══
camara = cv2.VideoCapture(0)
if not camara.isOpened():
    print("Error: no se pudo abrir la camara.")
    exit()

print("Webcam abierta. Pulsa 'q' para salir.")

# ═══ Variables que guardan el ultimo resultado ═══
contador = 0
nombre_actual = "?"
emocion_actual = "?"
edad_actual = "?"
x, y, ancho, alto = 0, 0, 0, 0
hay_cara = False

# ═══ BUCLE PRINCIPAL ═══
while True:
    ret, frame = camara.read()
    if not ret:
        break

    contador += 1

    # ─── Cada X frames: identificar + analizar ───
    if contador % FRECUENCIA == 0:

        # PASO 1: Buscar quien es en la base de datos
        try:
            resultados_find = DeepFace.find(
                img_path=frame,
                db_path=CARPETA_BD,
                enforce_detection=False,
                silent=True
            )
            if len(resultados_find) > 0 and len(resultados_find[0]) > 0:
                mejor = resultados_find[0].iloc[0]
                ruta = mejor["identity"]
                nombre_actual = os.path.splitext(os.path.basename(ruta))[0]
            else:
                nombre_actual = "Desconocido"
        except Exception:
            nombre_actual = "?"

        # PASO 2: Analizar emocion y edad
        try:
            resultados_analisis = DeepFace.analyze(
                img_path=frame,
                actions=["emotion", "age"],
                enforce_detection=False,
                silent=True
            )
            if len(resultados_analisis) > 0:
                cara = resultados_analisis[0]
                emocion_actual = cara.get("dominant_emotion", "?")
                edad_actual = cara.get("age", "?")
                region = cara.get("region", {})
                x = region.get("x", 0)
                y = region.get("y", 0)
                ancho = region.get("w", 0)
                alto = region.get("h", 0)
                hay_cara = True
            else:
                hay_cara = False
        except Exception:
            hay_cara = False

    # ─── Dibujar resultados sobre el video ───
    if hay_cara:
        # Color: verde si esta identificado, rojo si es desconocido
        if nombre_actual not in ("Desconocido", "?"):
            color = (0, 200, 0)    # Verde
        else:
            color = (0, 0, 200)    # Rojo

        # Rectangulo alrededor de la cara
        cv2.rectangle(frame, (x, y), (x + ancho, y + alto), color, 2)

        # Nombre encima del rectangulo
        cv2.putText(frame, nombre_actual, (x, y - 30),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.8, color, 2)

        # Emocion y edad debajo del nombre
        cv2.putText(frame, f"{emocion_actual}, {edad_actual} year", (x, y - 10),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.6, color, 2)

    # ─── Mostrar el frame en la ventana ───
    cv2.imshow("DeepFace Webcam", frame)

    # ─── Teclas ───
    tecla = cv2.waitKey(1) & 0xFF
    if tecla == ord("q"):
        break
    if tecla == ord("s"):
        cv2.imwrite(f"captura_{contador}.jpg", frame)
        print(f"Foto guardada: captura_{contador}.jpg")

# ═══ Limpieza al salir ═══
camara.release()
cv2.destroyAllWindows()
print("Webcam cerrada.")

Explicacion del código

🧠 ¿Por quédos llamadas a DeepFace?

Este programa hace dos cosas diferentes cada vez que analiza un frame:

  • DeepFace.find() — Busca la cara en la carpeta base_datos y nos dice quién es (el nombre del archivo mas parecido).
  • DeepFace.analyze() — Analiza la cara y nos dice su emoción y edad.

Son funciónes distintas porque hacen cosas distintas: find() compara contra una base de datos, analyze() extrae atributos. Combinándolas, conseguimos el resultado completo: nombre + emoción + edad.

FRECUENCIA = 45
Cada cuántos frames se ejecuta el analisis. Como ahora hacemos dos llamadas a DeepFace (find + analyze), necesitamos un poco mas de tiempo entre cada analisis. Con 45 a 30fps, analizamos cada ~1.5 segúndos. Si tu ordenador es rápido, baja a 30. Si va lento, sube a 60.
DeepFace.find()
Busca la cara del frame en todas las fotos de la carpeta base_datos. Devuelve la coincidencia mas parecida. La primera vez tarda mas porque genera un archivo de caché con los "números" (embeddings) de cada cara. Las siguientes veces es mucho mas rápido.
os.path.splitext(os.path.basename(ruta))[0]
De una ruta como "base_datos/Carlos.jpg", extrae solo el nombre: "Carlos". Primero basename() quita la carpeta ("Carlos.jpg"), luego splitext() separa el nombre de la extensión, y [0] coge solo el nombre.
nombre_actual, emoción_actual, edad_actual
Estas variables guardan el ultimo resultado del analisis. Se actualizan cada X frames pero se dibujan en cada frame. Por eso la etiqueta se mantiene visible entre analisis, en vez de parpadear.
Color verde / rojo
Si la persona esta en la base de datos, el rectangulo y el texto se muestran en verde. Si es desconocido o no se pudo identificar, en rojo. Esto da un feedback visual inmediato.
Dos líneas de putText
Dibujamos el nombre en una línea (mas grande, a 0.8) y la emoción + edad en otra línea justo debajo (mas pequeña, a 0.6). Usamos y - 30 y y - 10 para que no se solapen.

def registrar_asistencia(nombre, archivo_csv):
    """Registra una asistencia evitando duplicados del mismo día."""
    ahora = datetime.now()
    fecha = ahora.strftime("%Y-%m-%d")
    hora = ahora.strftime("%H:%M:%S")

    # Ver quién ya se ha registrado hoy
    registrados_hoy = set()
    if os.path.exists(archivo_csv):

Explicacion del código

registrar_asistencia()
Funcion que recibe un nombre y lo apunta en el CSV con la fecha y hora. Antes de escribir, lee todo el archivo y comprueba si esa persona ya se registró hoy. Si ya existe, no la duplica. Esto evita que el sistema registre a la misma persona 100 veces mientras está delante de la cámara.
registrados_hoy = set()
Un set (conjunto) es como una lista pero sin elementos repetidos y mucho mas rápida para buscar. Recorremos cada línea del CSV, extraemos el nombre y la fecha, y si la fecha es hoy, añadimos el nombre al conjunto. Despues, comprobar si alguien ya está es instantaneo con if nombre not in registrados_hoy.
guardar_captura()
Guarda una foto del frame actual como evidencia. El nombre del archivo incluye el nombre de la persona y la fecha/hora exacta (Carlos_20240315_091523.jpg), así cada foto es unica y sabes exactamente cuándo se tomó.
contador % 60
El sistema identifica personas cada 60 frames (~2 segúndos a 30fps). DeepFace.find() es mas lento que analyze() porque tiene que comparar contra toda la base de datos. Si lo ejecutáramos en cada frame, el video se congelaria. Puedes ajustar este número: menor = mas reactivo pero mas lento.
DeepFace.find(img_path=frame)
Busca la cara del frame actual en todas las fotos de la carpeta base_datos. Devuelve la coincidencia mas cercana. Pasamos directamente el frame (array NumPy) en vez de una ruta de archivo, lo que permite analizar video en tiempo real.
color condicional
Si la persona esta identificada (nombre distinto de "Desconocido"), el texto se muestra en verde (0, 200, 0). Si no se reconoce, en rojo (0, 0, 200). Esto da feedback visual inmediato: verde = todo bien, rojo = persona no registrada.
datetime.now().strftime()
Obtiene la fecha y hora actual del sistema y la formatea como texto. "%d/%m/%Y %H:%M:%S" produce por ejemplo "15/03/2024 09:15:23". Se muestra en la esquina de la ventana (cómo las cámaras de seguridad) y se usa para registrar la hora exacta de asistencia.

model_name
Le dice a DeepFace qué "cerebro" usar para reconocer caras. Cada modelo fue entrenado de forma diferente y da resultados ligeramente distintos. "VGG-Face" es el de por defecto. "ArcFace" es el más preciso en competiciones profesionales. "SFace" es el más rápido sin perder mucha precisión.
detector_backend
Le dice a DeepFace qué algoritmo usar para encontrar dónde están las caras (diferente del modelo que las reconoce). Es como la diferencia entre "buscar un objeto en una habitación" (detector) y "saber qué objeto es" (modelo).
🧠 ¿Modelo o detector? ¿Cuál es la diferencia?

Son dos pasos diferentes que trabajan juntos:

  • Detector (detector_backend) → "¿Dónde hay una cara en esta imagen?" → Devuelve coordenadas (x, y, ancho, alto).
  • Modelo (model_name) → "¿De quién es esta cara?" → Convierte la cara en 128 números y los compara.

Puedes combinar cualquier detector con cualquier modelo. Por ejemplo: retinaface (detector muy preciso) + ArcFace (modelo muy preciso) = la combinación más potente pero más lenta.

Como funcióna (paso a paso)

<script src="...face-api.js">
Carga la biblioteca face-api.js directamente desde internet (un CDN). Es como hacer import DeepFace en Python, pero en JavaScript lo haces con una etiqueta <script> que apunta a una URL. No necesitas descargar nada manualmente.
async / await
En JavaScript, las operaciones lentas (descargar modelos, encender webcam) son asincronas: el programa no se queda esperando, sigue ejecutando otras cosas. async marca una función que contiene operaciones lentas, y await le dice "espera aquí hasta que termine". Es parecido a como funcióna Python normalmente, pero en JavaScript hay que escribirlo explicitamente.
loadFromUri()
Descarga los modelos de IA desde la URL. Son archivos de ~1-3 MB cada uno. La primera vez tarda unos segúndos (dependiendo de la velocidad de internet). Las siguientes veces, el navegador los carga desde su cache y es casí instantaneo.
getUserMedia({ video: true })
Le pide al navegador acceso a la webcam. Chrome mostrara un aviso: "Este sitio quiere usar tu cámara". El alumno debe hacer clic en "Permitir". Si lo deniega, la webcam no se enciende.
setInterval(detectar, 500)
Ejecuta la función detectar cada 500 milisegúndos (cada medio segúndo). Es el equivalente al contador % 30 que usábamos en Python. Puedes cambiar el número: 300 = mas rápido, 1000 = mas lento pero mas fluido.
detectAllFaces(video)
El equivalente de DeepFace.analyze() en Python. Busca todas las caras en el frame actual del video. Los metodos encadenados (.withFaceLandmarks(), .withFaceExpressions(), .withAgeAndGender()) le dicen que informacion queremos de cada cara.
canvas
Es una capa transparente que se coloca encima del video. Dibujamos los rectangulos y etiquetas en el canvas, no en el video. Así el video sigue fluyendo debajo y los dibujos se actualizan sin afectar la imagen. Es como poner un cristal encima de una pantalla y dibujar con rotulador.
expressions.asSortedArray()[0]
face-api.js devuelve las 7 emociones con su probabilidad. asSortedArray() las ordena de mayor a menor, y [0] coge la primera = la emoción dominante. Es similar a cara["dominant_emotion"] en DeepFace.
💡 Python vs JavaScript: guía rápida de traducción
  • resultado = ... (Python) → const resultado = ... (JavaScript)
  • f"texto {variable}" (Python) → `texto ${variable}` (JavaScript, con acentos graves)
  • print(...) (Python) → console.log(...) (JavaScript)
  • for cara in resultados: (Python) → for (const cara of resultados) {} (JavaScript)
  • def función(): (Python) → function función() {} (JavaScript)

Explicacion de las partes clave

faceRecognitionNet
Este es el modelo que genera los 128 números (descriptor) que identifican una cara. Es el modelo equivalente a lo que hace DeepFace.find() en Python. Sin este modelo, podríamos detectar caras pero no saber de quien son.
detectSingleFace().withFaceDescriptor()
Detecta UNA cara y calcula su descriptor (los 128 números). Usamos detectSingleFace en vez de detectAllFaces porque al registrar solo necesitamos la cara de la persona que está delante de la cámara.
Array.from(det.descriptor)
El descriptor es un Float32Array (un tipo especial de JavaScript). Para guardarlo en localStorage necesitamos convertirlo a un array normal con Array.from(), porque JSON.stringify() no funcióna directamente con Float32Array.
localStorage
Es el "disco duro" del navegador. Guarda datos como texto (JSON) que persisten aunque cierres Chrome. Las personas registradas y las asistencias se guardan aqui. Tiene un limite de ~5 MB, suficiente para decenas de personas.
euclideanDistance()
Calcula la distancia entre dos descriptores de cara (igual que en la sección 6). Si la distancia es menor que 0.6, face-api.js considera que son la misma persona. Este umbral funcióna bien para la mayoria de casos.
lista.some(a => ...)
Comprueba si alguno de los registros cumple una condicion. Aquí verificamos si ya hay una asistencia con el mismo nombre y la misma fecha. Si ya existe, no se registra otra vez. Es el equivalente al if nombre not in registrados_hoy de la versión Python.
Blob + createObjectURL
Para descargar el CSV, creamos un Blob (un archivo en memoria) con el contenido CSV, generamos una URL temporal, y simulamos un clic en un enlace de descarga. Es la forma estandar de descargar archivos generados por JavaScript sin necesitar un servidor.

Ética e IA responsable

⚠️ Marco legal

En Europa, el RGPD clasifica los datos faciales como datos biométricos (categoría especial). Usar reconocimiento facial sin consentimiento puede ser ilegal y conlleva multas muy elevadas.

1. Privacidad: ¿Quién tiene derecho a grabar y analizar tu cara? ¿Hay cámaras con reconocimiento facial en tu barrio?

2. Sesgos: Si un modelo funcióna peor con ciertos grupos étnicos, eso no es solo un "error técnico" — es un problema de discriminación.

Detector de mascarillas

📊
Mapa de emociones de clase
Haz una foto de grupo y analiza la emoción de cada persona. Genera un grafico de barras con matplotlib (Python) o Chart.js (JavaScript) que muestre el "estado de animo" general de la clase.
3
Sistema de acceso seguro
🚪
Conecta el reconocimiento facial a una Raspberry Pi con un servo motor. Si la persona esta en la base de datos → el servo abre una puerta. Un proyecto completo de electronica + IA.
4
🎮
Juego controlado con gestos
Usa la detección de emociones para controlar un juego: sonrie para saltar, abre la boca para disparar, levanta las cejas para pausar. Combina face-api.js con un canvas de juego HTML.
